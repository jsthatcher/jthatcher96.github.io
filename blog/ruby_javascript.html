Ruby Classes vs Javascript Constructor Functions

3/28/2016

Ruby classes and Javascript constructor functions, are they the same? Yes, no, and a little bit. The general idea behind these two similiar structures is to be able to replicate an empty object. Though these objects are empty they still have proporties that exist in each instance of the structure. A situation where this might be helpfull would be if you needed to store the information you find on a drivers lisinse for mulitple people. Instead of creating a variable for each peice of information for each person. You could make a structure with the skeleton of the information that you needed, then for each person you would just call an instance of the structure and fill in the appropriate information.
